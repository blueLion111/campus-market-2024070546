## 遇到的问题

### 问题 1：目录结构过于简单，难以理解分层意图
**现象**：初始看到 `api/`、`components/`、`stores/` 等目录中只有 `.gitkeep` 文件，不理解这些目录的作用。

**解决过程**：
1. 先阅读 `package.json` 了解项目依赖
2. 查看 `main.ts` 和 `router/index.ts` 理解框架工作方式
3. 对比 Vue 官方文档的项目结构说明
4. 结合各目录的命名，推断其功能定位

**经验总结**：种子项目的 `.gitkeep` 文件是一种设计模式，提示开发者"此处待实现"。应结合配置文件和已有代码推断目录功能。

### 问题 2：对 Router 的 History 模式理解模糊
**现象**：`createWebHistory` 和 `createWebHashHistory` 的区别不清晰。

**解决过程**：
1. 阅读 vue-router 官方文档
2. 理解 History API 和 Hash 的工作原理
3. 实践：访问带 `#` 和不带 `#` 的 URL，观察浏览器行为差异

**理解记忆**：
- History 模式：`/home`，需要服务器配置，URL 更美观
- Hash 模式：`/#/home`，无需服务器配置，兼容性更好

### 问题 3：pnpm 包管理器不熟悉
**现象**：习惯使用 npm，pnpm 的命令和配置有差异。

**解决过程**：
1. 查看 `package.json` 中的 `packageManager` 字段
2. 查阅 `.nvmrc` 了解 Node 版本要求
3. 对比 npm 与 pnpm 的常用命令差异

**经验总结**：pnpm 的 `pnpm install` 与 npm 的 `npm install` 功能类似，但 pnpm 使用硬链接，节省磁盘空间且安装更快。

## AI 协作情况

### 协作场景

| 场景 | AI 提问 | AI 输出价值 |
|------|---------|-------------|
| 项目结构分析 | "分析项目目录结构" | 输出了完整的目录树和对各目录功能的解释 |
| 代码解释 | "解释 main.ts 的作用" | 逐行分析代码，说明每个 API 的用途 |
| 概念学习 | "什么是 Vue Router" | 解释了路由的核心概念和工作流程 |

### 协作效果评估

**优点**：
- AI 能够快速分析大量代码，提供结构化的解释
- 节省了查阅文档的时间，尤其是对于陌生的技术栈
- 输出了格式良好的文档结构，减少了排版工作

**局限**：
- AI 的解释基于通用理解，可能不适用于特定项目场景
- 需要结合源码验证 AI 的输出是否准确
- AI 无法替代动手实践，代码能力提升需要实际编码

**改进方向**：
- 提问时提供更多上下文，帮助 AI 输出更精准的答案
- 对 AI 输出保持批判性思考，结合源码验证
- 记录 AI 的局限性，形成个人知识体系
